Wolfram Language XML

A syntax extension for Wolfram Language that lets you write HTML-like markup inside a Wolfram Language Script like JSX.

Getting started - 5min ⏱️
Heading[Text_] := <h2 class="tracking-tight"><Text/></h2>;

<body>
  <Heading>
    Hello World!
  </Heading>
</body>

Write components with code and markup

HTML is simple, human-readable and straighforward. Having an XML nature it allows to use Wolfram Expressions as tags.

Regular WL expression
TextString[Now]
WLX expression
<TextString>
  <Now/>
</TextString>

Features of WL are preserved

First child element is the first argument of a given function and etc. Any self-closing tags represent an own-value of a symbol. Outside XML tags any WL expressions are valid.

ClassName = "show";
<div class="{ClassName} otherClass" style="color: cyan">
      Some text...
</div>

Passing HTML/XML attributes

We declare a variable called ClassName and then use it inside WLX by wrapping it in curly braces. Unlike JSX it does not restric you on writting expression in this form. One can write a plain HTML with CSS along with WLX.

Columns = Table[
    <div class="lg:pr-4">
      <div class="text-base leading-7 text-gray-700 ">
        <Child/>
      </div>
    </div>
, {Child, $Children}]
  
<div class="row">
  <Columns/>
</div>

WLX expression is Wolfram Language expression too

HTML/XML is a markup language by its nature. Therfore it is recommended not to use explicitly Table or If expressions inside XML tags, but rather utilize Wolfram Language for that.

main.wlx
Header = ImportComponent["header.wlx"];
(* /* use it as WL expression */ *)
<body>
  <Header Title={"WLX is awesome"} />
</body>
header.wlx
<h1>
  <Title/>
</h1>

Components aproach

All components or wlx scripts are scoped, i.e. defined variables inside a component will not leak to the global scope.

plotter.wlx
Data = Drop[Import[$FirstChild, "TSV"], 3];

PlottingDevice = ListLinePlot[Data, PlotRange->Full, ImageSize->700];

<figure style="display: inline-block">
    <WLJS Class={"h-auto max-w-full flex rounded-lg p-3 bg-gray-100"}>
        <PlottingDevice/>
    </WLJS>
    <figcaption class="text-center gap-x-4 min-w-0 mt-1 text-xs leading-5 text-gray-500">Drag - pan, wheel - zoom</figcaption>
</figure>

WLJS and all features of frontend are supported

All components or wlx scripts are scoped, i.e. defined variables inside a component will not leak to the global scope.

text     = "nothing";
View     = TextView[Offload[text]] // WLJS;
Button   = ButtonView["Press me", "Event"->Secret] // WLJS; 

EventHandler[Secret, Function[void, text = RandomWord[]]];

<div>
    <View/>
    <Button/>
</div>
