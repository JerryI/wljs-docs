Interpreter

WLJS interpreter is a standalone library, that can be used in a web applications as well. Since it is a core element of WLJS Frontend, all features like plotting, graphics, even some live computations are possible as well. To unlock full potential see dynamics section.

Firstly, one need to add a helper structure to add all necessary scripts

<<JerryI`WLX`WLJS`
note

It is assumed you have read an installation guide install

A shortcut
If you have git installed. Simply clone
git clone https://github.com/JerryI/wl-wlx
cd wl-wlx
wolframscript -f Examples/WLJSBasic.wls

that will run the simplest example possible

Available JS libraries​

Then, in the <head> of your HTML document you need to put links to the necessary javascript libraries you want to use

<WLJSHeader>
    https://cdn.statically.io/gh/JerryI/wljs-interpreter/main/src/interpreter.js
    https://cdn.statically.io/gh/JerryI/wljs-interpreter/main/src/core.js
    https://cdn.statically.io/gh/JerryI/wljs-graphics-d3/main/dist/kernel.js
    https://cdn.statically.io/gh/JerryI/wljs-plotly/main/dist/kernel.js
    https://cdn.statically.io/gh/JerryI/wljs-inputs/main/dist/kernel.js
    https://cdn.statically.io/gh/JerryI/Mathematica-ThreeJS-graphics-engine/master/dist/kernel.js
</WLJSHeader>

Here statically.io CDN is used, but jsdelivr is also fine. The list of all core packages available one can find in the repo of @JerryI with a prefix wljs-*.

info

It is important to include firstly interpreter.js and then core.js. An order of them matters. Usually files are located in dist directory of a package, if there is none, use src folder.

The list of the most essential libraries is following

  • wljs-interpreter interpreter & core library & meta-markers
  • wljs-graphics-d3 2D graphics
  • wljs-plotly Alternative 2D plots using Plotly.js
  • wljs-graphics3d-threejs 3D graphics
  • wljs-inputs UI elements
note

Since WLJS Frontend actually provides some helper functions, one should use those libraries carefully, because not all features listed can be used without a real frontend.

Usage​

An access to the interpreter can be done by wrapping Wolfram Expressions via special tag or function

P = Plot[x, {x, 0, 1}];
<WLJS><P/></WLJS>

What it does it exports the result of the evaluation to ExpressionJSON and embeds it to the page, where the tag is located. Lately when the page was loaded, a javascript code will be executed. If there is any graphics output presented it will embed it directly to the <div> element.

note

An expression will be evaluated inside a container attached to the <div> element via AttachDOM method placed on the location of <WLJS> tag.

note

You can also combine WLJS code with your custom Javascript functions. See how architecture

Or even more complicated stuff

VectorPlot = VectorPlot3D[{x, y, z}, {x, -1, 1}, {y, -1, 1}, {z, -1, 1}, 
 VectorColorFunction -> Function[{x, y, z, vx, vy, vz, n}, ColorData["ThermometerColors"][x]]
];

<WLJS>
    <VectorPlot/>
<WLJS/>

Styling​

One can pass any class names (from CSS table) to <WLJS> tag to customize the output

<style>
    .myClass {
        position:absolute;
        top: 0;
        left: 0;
        right: 0;
        bottom: 0;
        margin: auto;
    }
</style>

<WLJS Class={"myClass"}><P/></WLJS>
